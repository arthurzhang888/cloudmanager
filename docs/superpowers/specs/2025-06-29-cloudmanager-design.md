# CloudManager 系统设计文档

## 1. 系统概述

### 1.1 目标
构建一个企业级数据中心基础设施管理平台，支持多地大型数据中心（1000+ 服务器）的统一管理，具备可组合基础设施能力和液冷系统深度集成。

### 1.2 核心定位
- **统一管控平面**：单界面管理多地数据中心
- **可观测性优先**：资产发现 → 监控 → 编排的渐进演进
- **液冷原生**：从第一天就支持 CDU/冷却塔的传感器采集和 PUE 优化

## 2. MVP 阶段功能范围（2-3 个月）

### 2.1 边缘 Agent（Go）
部署在每个数据中心，负责本地硬件发现和数据采集。

| 模块 | 功能 |
|-----|------|
| **硬件发现** | Redfish 扫描服务器、SNMP 发现网络设备 |
| **液冷采集** | Modbus TCP/RTU 读取 CDU 和冷却塔传感器 |
| **数据上报** | 通过 gRPC/mTLS 向云端推送遥测数据 |
| **离线缓存** | 网络中断时本地缓存，恢复后批量上报 |

### 2.2 云端后端

| 服务 | 职责 |
|-----|------|
| **API Gateway** | 统一入口、认证鉴权、流量控制 |
| **Asset Service** | 资产元数据管理、生命周期追踪 |
| **Telemetry Service** | 遥测数据接收、PUE 计算、告警触发 |
| **Discovery Service** | 管理边缘 Agent 的发现任务 |

### 2.3 数据存储

| 存储 | 用途 |
|-----|------|
| **PostgreSQL** | 资产信息、用户、配置、拓扑关系 |
| **TimescaleDB** | 时序数据（温度、功耗、PUE 历史）|
| **Redis** | 实时状态缓存、会话、分布式锁 |

### 2.4 Web 控制台（React + TypeScript）

| 功能 | 描述 |
|-----|------|
| **资产总览** | 服务器/网络设备/液冷设备清单 |
| **拓扑视图** | 机架级拓扑、数据中心级拓扑 |
| **液冷监控** | CDU/冷却塔实时数据、PUE 仪表盘 |
| **告警中心** | 阈值告警、设备离线告警 |

## 3. 技术架构详情

### 3.1 边缘 Agent 架构

```
edge-agent/
├── discovery/        # 硬件发现
│   ├── redfish.go    # Redfish 客户端
│   ├── snmp.go       # SNMP 扫描
│   └── scanner.go    # 发现调度
├── collector/        # 数据采集
│   ├── modbus.go     # Modbus TCP/RTU 客户端
│   ├── redfish_metrics.go
│   └── scheduler.go  # 采集调度
├── uploader/         # 数据上报
│   ├── grpc_client.go
│   └── buffer.go     # 离线缓存
└── config/           # 本地配置管理
```

**Modbus 采集参数（CDU）：**
- 采集频率：30 秒（可配置）
- 数据点：进水温度、回水温度、流量、压力、泵状态、PUE 相关参数
- 寄存器映射：通过配置适配不同厂商 CDU

### 3.2 云端服务架构

```
cloud-backend/
├── api-gateway/      # HTTP/gRPC 网关
├── asset-service/    # 资产管理
├── telemetry-service/ # 遥测处理
├── discovery-service/ # 发现任务管理
└── shared/
    ├── models/       # 数据模型
    └── db/           # 数据库连接
```

### 3.3 网络安全设计

**混合网络适配：**
- **专线/VPN 环境**：直接使用 gRPC 长连接，双向通信
- **公网环境**：gRPC over mTLS，边缘 Agent 主动连接云端（出站 443），支持连接池和断线重连

**证书管理：**
- 自建 CA 签发证书
- 边缘 Agent 首次启动时通过 bootstrap token 获取证书
- 证书自动轮换（90 天有效期）

### 3.4 数据模型（核心实体）

```sql
-- 数据中心
data_centers (id, name, location, network_type)

-- 边缘 Agent
edge_agents (id, dc_id, hostname, version, last_seen, cert_sn)

-- 服务器
servers (
    id, agent_id, redfish_endpoint, manufacturer, model,
    serial_number, cpu_count, memory_gb, status, discovered_at
)

-- 液冷设备
cooling_devices (
    id, agent_id, device_type,  -- 'cdu' | 'cooling_tower'
    modbus_address, name, location
)

-- 传感器数据（TimescaleDB hypertable）
sensor_readings (
    time, device_id, metric_name, value, unit
)

-- 资产拓扑
topology_edges (parent_id, child_id, edge_type)
```

## 4. 关键流程

### 4.1 服务器发现流程

1. Agent 启动或定时触发
2. 扫描配置的 IP 段（Redfish 默认端口 443）
3. 对响应的设备进行 Redfish 详查（/Systems/1, /Chassis/1）
4. 上报设备信息到云端 Asset Service
5. 云端更新资产库，触发变更通知

### 4.2 液冷数据采集流程

1. Agent 从云端获取采集配置（CDU 列表、寄存器映射）
2. 每 30 秒轮询 Modbus 设备
3. 解析原始数据（温度 × 0.1°C，压力 × 0.01 bar 等）
4. 本地缓冲 → 批量上报 Telemetry Service
5. 云端写入 TimescaleDB，实时计算 PUE

### 4.3 PUE 计算

```
PUE = 总设施能耗 / IT 设备能耗

数据源：
- IT 能耗：服务器电源传感器（Redfish PowerControl）
- 设施能耗：CDU 功耗 + 冷却塔功耗（Modbus 电能寄存器）
```

## 5. 技术选型

| 组件 | 选择 | 理由 |
|-----|------|------|
| **Go** | 边缘 + 后端 | 低资源占用、原生并发、静态编译易部署 |
| **gRPC** | 边缘-云端通信 | 高性能、强类型、流式支持 |
| **NATS** | 消息队列（MVP） | 轻量、易部署，适合 MVP 阶段 |
| **TimescaleDB** | 时序数据 | PostgreSQL 兼容、自动分区、优秀的压缩率 |
| **React** | 前端 | 生态成熟、仪表盘组件丰富 |

## 6. 里程碑规划

| 阶段 | 时间 | 功能 | 交付物 |
|-----|------|------|--------|
| **MVP** | 2-3 个月 | 资产发现、液冷监控、基础仪表盘 | 可运行的管理平台，支持 1-2 个数据中心 |
| **v1.1** | +1 个月 | 告警规则引擎、历史趋势分析 | 完整的监控告警能力 |
| **v1.2** | +2 个月 | 资源池化、裸金属分配 | 基础可组合能力 |
| **v2.0** | +3 个月 | 完整可组合架构、多租户、高级编排 | 生产级平台 |

## 7. 风险评估与缓解

| 风险 | 缓解措施 |
|-----|---------|
| Modbus 设备兼容性 | 抽象采集层，支持寄存器映射配置 |
| 大规模并发 | 边缘 Agent 本地聚合，批量上报 |
| 网络不稳定 | 本地缓存 + 断点续传机制 |
| 证书管理复杂度 | 自动化证书签发和轮换 |

## 8. 系统边界

### 8.1 包含在 MVP 内
- 服务器资产发现（Redfish）
- 液冷系统监控（Modbus）
- 基础网络设备发现（SNMP）
- 资产清单和拓扑展示
- PUE 实时计算和展示
- 基础告警（阈值、离线）

### 8.2 不包含在 MVP 内（后续版本）
- 资源池化和动态分配
- 裸金属自动化部署
- 高级编排策略
- 多租户隔离
- 计费系统
- 容量规划预测
